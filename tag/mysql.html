mysql mongodb 备份脚本

四 24 三月 2016

Filed under 运维

Tags mysql mongodb

一 准备:

automysqlbackup下载
automongobackup下载

automongobackup是模仿automysqlbackup项目写的，默认并不带配置文件，可以直接修改脚本里对应变量。

​

二 配置

automysqlbackup 

CONFIG_mysql_dump_username='user'
CONFIG_mysql_dump_password='passwd'
CONFIG_mysql_dump_host='localhost'
CONFIG_backup_dir='/backup/mysql'
CONFIG_multicore='yes'
CONFIG_multicore_threads=2
CONFIG_do_monthly="0"
CONFIG_do_weekly="0"
CONFIG_rotation_daily=6
CONFIG_mysql_dump_port=3306
CONFIG_mysql_dump_single_transaction='yes'
CONFIG_mysql_dump_compression='gzip'

automongobackup 

DBHOST="127.0.0.1"
DBPORT="27017"
BACKUPDIR="/backup/mongodb"
DBUSERNAME="user"
DBPASSWORD="passwd"
DBAUTHDB="admin"
MAILCONTENT="stdout"
MAXATTSIZE="4000"
DODAILY="yes"
DAILYRETENTION=6
DOWEEKLY="no"
WEEKLYDAY=0
WEEKLYRETENTION=0
DOMONTHLY="no"
MONTHLYRETENTION=0
COMP="gzip"
CLEANUP="yes"
LATEST="yes"
LATESTLINK="yes"
OPLOG="no"
REPLICAONSLAVE="yes"
REQUIREDBAUTHDB="yes"

两个配置都指定备份文件保存一星期。

三 执行

/usr/local/bin/automysqlbackup /etc/automysqlbackup/myserver.conf
automongobackup

四 数据导入

mongodb

mongorestore -h 127.0.0.1 -d dbname dbbakdir/

mysql

mysql -uusername -ppassword < bak.sql

mysql5.6使用老格式密码

三 23 三月 2016

Filed under 运维

Tags mysql

转发的，找不到原文了。。

SET SESSION old_passwords=0; SELECT PASSWORD('123456'); SET SESSION old_passwords=1; SELECT PASSWORD('123456');

实际操作： 登陆mysql，使用SET SESSION old_passwords=0; //设置老格式密码 SET PASSWORD FOR 'webuser'@'localhost' = PASSWORD('123456') //然后为用户设置密码。

修改配置文件： [client] secure_auth = 0 [mysqld] old_passwords=1

成功登陆！

忘记用户密码时： [mysqld] skip-grant-tables

然后不用口令进入mysql修改密码：

UPDATE mysql.user SET Password=PASSWORD ...
